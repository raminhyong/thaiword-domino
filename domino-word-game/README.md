# โดมิโนคำ (คำประสม-คำซ้อน) 🁫

เกมต่อคำแบบโดมิโน เล่นสดในห้องเรียน คล้าย Kahoot: จอโปรเจกเตอร์ (host) + จอมือถือผู้เล่น (player) รองรับสูงสุด 10 ทีมต่อห้อง และรองรับหลายห้อง/หลายครูเล่นพร้อมกันในเซิร์ฟเวอร์เดียว

## สรุปกติกาที่ระบบทำตาม

- คำทั้งหมด 354 ใบ (คำซ้อน 135 + คำประสม 189 + BLANK ไวลด์การ์ด 30) มาจาก `คลังคำ.docx` ของคุณ ตัดคำ 2 คำที่มี 3 พยางค์ออก (เครื่องแต่งกาย, นิ่งนอนใจ) เพราะกลไกโดมิโนต้องใช้คำ 2 พยางค์พอดี
- ต่อคำโดยจับคู่พยางค์ที่ตรงกันเป๊ะกับปลายเปิดของโซ่ (แบบโดมิโนจริง) วางปกติหรือกลับหัวได้ตามตัวอย่างที่คุณให้ (บ้านเรือน → เรือนหอ / ออกเรือน)
- โซ่ต่อยาวไปทางเดียวตลอดเกม
- เทิร์นวนตามลำดับทีม (สุ่มทีมแรกก่อน) เทิร์นละ 30 วินาที
- หมดเวลา = บังคับจั่ว 1 ใบเป็นโทษ จบเทิร์นทันที ไม่มีโอกาสลง
- กดจั่วเองระหว่างเทิร์นได้ (ไม่ใช่โทษ) แล้วลองลงคำที่จั่วได้ในเทิร์นเดียวกัน นาฬิกาเดินต่อรวม 30 วิ
- ลงผิด (พยางค์ไม่ตรง หรือ ลงผิดช่องคำซ้อน/คำประสม) = เด้งกลับ + โทษจั่วเพิ่ม 1 ใบ + จบเทิร์น
- เบี้ยเริ่ม 10 ใบ/ทีม ถาดมี 15 ช่อง ถ้าถาดเต็มแล้วยังต้องจั่วอีก (จนตรอกจริง) = ทีมนั้นถูกคัดออก
- BLANK ลงช่องไหนก็ได้ ไม่ต้องตรงพยางค์ และเปิดปลายโซ่เป็น "อะไรก็ได้" ให้ทีมถัดไปลงคำใดก็ได้ 1 ครั้ง
- จบเกมเมื่อหาทีมอันดับ 1-2-3 ได้ครบ (ทีมไหนเบี้ยหมดก่อนได้อันดับนั้น) ขึ้นแท่นรางวัลแล้วจบ
- กรณีพิเศษ: ถ้าทุกทีมที่เหลือถูกคัดออกหมดก่อนจะครบ 3 อันดับ (โชคร้ายทั้งกระดาน) ระบบจะจัดอันดับที่เหลือให้ทีมที่มีเบี้ยเหลือน้อยที่สุด (ใกล้ชนะที่สุด) เพื่อให้เกมจบแบบมีผลสรุปเสมอ

**ข้อสังเกตเรื่อง balance:** จากการทดสอบจำลอง มีโอกาสที่พยางค์บางตัวจะหายาก ทำให้บางเกมทุกทีมเสี่ยงโดนคัดออกพร้อมกันได้ (ก่อนใครจะชนะจริง) BLANK ช่วยแก้ตรงนี้ได้ระดับหนึ่งแต่ไม่ทั้งหมด ถ้าเล่นแล้วรู้สึกว่าเบี้ยเต็ม/ถูกคัดบ่อยเกินไป อาจพิจารณาเพิ่มจำนวน BLANK ในกองกลาง (แก้ที่ `data/wordbank.json` หรือบอกผมให้ปรับได้)

## โครงสร้างไฟล์

```
domino-word-game/
├── server.js           เซิร์ฟเวอร์ Express + Socket.IO (รองรับหลายห้องพร้อมกัน)
├── gameEngine.js        ตรรกะเกมทั้งหมด (server-authoritative)
├── data/wordbank.json    คลังคำ 354 ใบ
├── public/
│   ├── index.html        หน้าแรก เลือกโหมด host/player
│   ├── host.html/js/      จอโปรเจกเตอร์
│   ├── player.html/js/    จอมือถือผู้เล่น
│   └── styles.css         ธีมพาสเทลน่ารัก (แก้/เปลี่ยนได้ตามใจ)
└── test/
    ├── engine.test.js      ทดสอบตรรกะเกมล้วน ๆ (เร็ว ไม่ต้องเปิดเซิร์ฟเวอร์)
    └── integration.test.js ทดสอบผ่าน socket.io จริง (ต้องรันเซิร์ฟเวอร์ก่อน)
```

## รันทดสอบในเครื่องตัวเอง

```bash
npm install
npm test              # รัน engine.test.js
node server.js         # แล้วเปิด http://localhost:3000
```

จากนั้นเปิด `http://localhost:3000/host.html` ในแท็บนึง (จอโปรเจกเตอร์) และ `http://localhost:3000/player.html` ในแท็บ/มือถืออีกเครื่อง (สแกน QR หรือกรอกรหัสห้องเอง)

## จะ deploy ที่ไหนดี?

เกมนี้ต้องการ **การเชื่อมต่อค้างสาย (WebSocket)** ตลอดเวลาที่เล่น เพื่อส่งสัญญาณเทิร์น/นาฬิกา/การลงคำแบบเรียลไทม์ — **Netlify เพียงอย่างเดียวใช้ไม่ได้** เพราะ Netlify เป็นเว็บโฮสต์แบบ static site + serverless function ที่รันเป็นช่วงสั้น ๆ ไม่รองรับเซิร์ฟเวอร์ Node.js ที่ต้องเปิดค้างไว้แบบนี้

### แนะนำ: Render.com (ใกล้เคียง Netlify มากที่สุดในแง่ความง่าย)

ขั้นตอน:
1. อัปโหลดโค้ดทั้งโฟลเดอร์นี้ขึ้น GitHub (สร้าง repo ใหม่ แล้ว push ขึ้นไป)
2. ไปที่ https://render.com สมัคร/ล็อกอิน (เชื่อมกับ GitHub ได้เลย)
3. New + → Web Service → เลือก repo นี้
4. ตั้งค่า:
   - Build Command: `npm install`
   - Start Command: `npm start`
   - Instance Type: แนะนำ **Starter (~$7/เดือน)** เพื่อให้เซิร์ฟเวอร์ไม่ "หลับ" ระหว่างคาบเรียน (แผนฟรีของ Render จะ sleep เมื่อไม่มีคนใช้ และใช้เวลาปลุกตื่นหลายสิบวินาที ซึ่งจะกระทบตอนนักเรียนสแกน QR พร้อมกัน)
5. กด Deploy รอสักครู่ จะได้ลิงก์ เช่น `https://your-app.onrender.com`
6. เปิด `https://your-app.onrender.com/host.html` บนโปรเจกเตอร์ และบอกนักเรียนเข้า `https://your-app.onrender.com/player.html` (หรือสแกน QR ที่ระบบสร้างให้อัตโนมัติ ซึ่งจะฝัง URL นี้ให้เองตามโดเมนจริงที่ deploy)

ทางเลือกอื่น: Railway.app หรือ Fly.io ก็ใช้หลักการเดียวกัน (รัน Node.js เซิร์ฟเวอร์ค้างไว้) ราคาและขั้นตอนใกล้เคียงกัน เลือกได้ตามสะดวก

### ข้อจำกัดที่ควรรู้

- สถานะเกมทั้งหมดเก็บอยู่ใน **หน่วยความจำ (RAM)** ของเซิร์ฟเวอร์ ไม่ได้บันทึกลงฐานข้อมูล ถ้าเซิร์ฟเวอร์ต้อง restart (เช่น deploy ใหม่) ระหว่างเล่น เกมที่กำลังเล่นอยู่จะหายไป ต้องเริ่มห้องใหม่ — เหมาะกับการใช้เล่นสดต่อคาบเรียน ไม่ได้ออกแบบให้ค้างไว้ข้ามวัน
- รองรับหลายห้องพร้อมกันในเซิร์ฟเวอร์เดียวอัตโนมัติ (แต่ละห้องมีรหัส 5 ตัวอักษรของตัวเอง) ไม่ต้องตั้งค่าอะไรเพิ่ม
- การลากคำบนมือถือใช้ Pointer Events (รองรับทั้งนิ้วสัมผัสและเมาส์) ยังไม่ได้ผูกกับ library ลากวางสำเร็จรูป ถ้าต้องการเอฟเฟกต์ลื่นขึ้นแจ้งได้
- ภาพ/ธีมตอนนี้เป็นพาสเทลเรียบ ๆ + อีโมจิ ไว้ให้ใช้งานได้จริงก่อน พร้อมให้คุณเปลี่ยนภาพเองทีหลังตามที่บอกไว้ (ไฟล์ที่เกี่ยวข้องคือ `public/styles.css` และตำแหน่ง emoji ใน `public/host.js`, `public/player.js`)

## ถ้าอยากให้ผมช่วยต่อ

บอกได้เลยเช่น: ปรับจำนวน BLANK, เพิ่มเอฟเฟกต์เสียง/แอนิเมชัน, ทำหน้าแอดมินควบคุมเกม (ข้ามเทิร์น/รีเซ็ต), รองรับผู้เล่นหลุดเน็ตให้กลับมาเล่นต่อได้ลื่นขึ้น, หรือช่วย push ขึ้น GitHub/ตั้งค่า Render ให้
